# <a name="defender-for-iot-glossary"></a>用于 IoT 的 Defender 术语表

本词汇表简要介绍了用于 IoT 平台的 Azure Defender 的重要术语和概念。 选择 " **了解更多** " 链接，以指向术语表中的相关术语。 这将帮助你更快速地学习和使用产品工具。

## <a name="a"></a>A

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **访问组** | 通过创建访问组规则，支持大型组织的用户访问要求。<br /><br />规则可让你控制针对相关业务部门、区域、站点和区域中特定用户角色的针对 IoT 本地管理控制台的访问权限。<br /><br />例如，允许来自 Active Directory 组的安全分析人员访问西欧的汽车数据，但阻止访问非洲的数据。 | **本地管理控制台 <br /> <br /> 业务部门** |
| **访问令牌** | 生成访问令牌以访问用于 IoT 的 Defender REST API。 | **API** |
| **确认警报事件** | 指示 Defender for IoT 为检测到的事件隐藏一次警报。 如果再次检测到该事件，将再次触发此警报。 | **警报 <br /> <br /> 了解警报事件 <br /> <br /> 静音警报事件** |
| **发出** | 与已授权网络行为、网络异常或可疑网络活动和流量的偏差有关的信息。 | **转发规则 <br /> <br /> 排除规则 <br /> <br /> 系统通知** |
| **警报注释** | 安全分析人员和管理员在警报消息中提出的注释。 例如，警报注释可能会提供有关要采取的缓解措施的说明，或与事件相关的人员姓名。<br /><br />查看警报的用户可以选择最能反映事件状态的注释或备注，也可以选择调查警报所采取的步骤。 | **发出** |
| **异常引擎** | 用于检测异常计算机到计算机 (M2M) 通信和行为的 IoT 引擎 Defender。 例如，引擎可能检测到过多的 SMB 登录尝试。 检测到这些事件时，将触发异常警报。 | **用于 IoT 引擎的 Defender** |
| **API** | 允许外部系统访问 Defender 为 IoT 发现的数据，并通过 SSL 连接使用外部 REST API 执行操作。 | **访问令牌** |
| **攻击向量报告** | 可利用终结点的漏洞链的实时图形表示。<br /><br />报表允许您评估攻击序列中缓解活动的影响。 例如，你可以通过中断攻击链来评估系统升级是否会破坏攻击者的路径，或者是否保留备用攻击路径。 这会排定修正和缓解活动的优先级。 | **风险评估报表** |

## <a name="b"></a>B

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **业务部门** | 根据特定行业，业务的逻辑组织。<br /><br />例如，包含玻璃工厂和塑料工厂的全球性公司可以作为两个不同的业务单位进行管理。 可以控制对 IoT 用户访问特定业务部门的 Defender。 | **本地管理控制台 <br /> <br /> 访问组 <br /> <br /> 站点 <br /> <br /> 区域** |
| **线** | 批准的网络流量、协议、命令和设备。 用于 IoT 的 Defender 标识与网络基线之间的偏差。 通过生成数据挖掘报表来查看已批准的基线流量。 | **数据挖掘 <br /> <br /> 学习模式** |

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **本地管理控制台** | 本地管理控制台提供了对设备和威胁的集中视图和管理，在你的组织中检测到 IoT 传感器部署的 Defender。 | **用于 IoT 平台传感器的 Defender <br /> <br />** |
| **CLI 命令** | 命令行接口 (用于 IoT 管理员用户的 Defender) 选项。 CLI 命令适用于无法从用于 IoT 控制台的 Defender 访问的功能。 | - |


## <a name="d"></a>D

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **数据挖掘** | 生成有关网络设备的全面的精细报表：<br /><br />- **SOC 事件响应**：实时报告以帮助处理即时事件响应。 例如，报表可以列出可能需要修补的设备。<br /><br />- **辩论**：基于调查报表的历史数据的报表。<br /><br />- **IT 网络完整性**：有助于提高总体网络安全的报表。 例如，报表可以列出具有弱身份验证凭据的设备。<br /><br />- **可见性**：涵盖所有查询项以查看网络的所有基线参数的报表。<br /><br />为只读用户保存数据挖掘报表以查看。 | **基线 <br /> <br /> 报表** |
| **用于 IoT 引擎的 Defender** | 用于 IoT 的自助分析引擎无需更新签名或定义规则。 这些引擎使用特定于 ICS 的行为分析和数据科学连续分析网络流量，以应对异常、恶意软件、操作问题、违反协议以及与基线网络活动的偏差。<br /><br />当引擎检测到偏差时，会触发警报。 可以从 " **警报** " 屏幕或从 SIEM 查看和管理警报。 | **发出** |
| **用于 IoT 平台的 Defender** | 在用于 IoT 传感器的 Defender 和本地管理控制台上安装了用于 IoT 的 Defender 解决方案。 | **传感器 <br /> <br /> 本地管理控制台** |
| **设备映射** | 用于 IoT 检测的网络设备的图形表示。 它显示设备与每个设备的相关信息之间的连接。 使用以下映射：<br /><br />-检索和控制关键设备信息。<br /><br />-分析网络切片。<br /><br />-导出设备详细信息和摘要。 | **Purdue 层组** |
| **设备清单-传感器** | 设备清单显示 Defender 用于 IoT 检测到的范围广泛的设备属性。 可用选项如下：<br /><br />-筛选显示的信息。<br /><br />-将此信息导出到 CSV 文件。<br /><br />-导入 Windows 注册表详细信息。 | **组设备清单-本地管理控制台** |
| **设备清单-本地管理控制台** | 可以从设备清单中的本地管理控制台查看已连接传感器的设备信息。 这为本地管理控制台的用户提供了所有网络信息的综合视图。 | **设备清单-传感器 <br /> <br /> 设备清单-数据集成器** |
| **设备清单-数据集成器** | 利用本地管理控制台的数据集成功能，你可以利用其他企业资源的信息来增强设备清单中的数据。 示例资源包括 CMDBs、DNS、防火墙和 Web Api。 | **设备清单-本地管理控制台** |

## <a name="e"></a>E

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **企业视图** | 一个全局地图，其中显示了安装了 IoT 传感器的 Defender 的业务部门、站点和区域。 查看恶意警报、操作警报等的地理位置。 | **业务部门 <br /> <br /> 网站 <br /> <br /> 区域** |
| **事件时间线** | 在您的网络上检测到的活动的时间线，其中包括：<br /><br />-触发警报。<br /><br />- (信息性) 的网络事件。<br /><br />-用户操作（例如登录、用户删除和用户创建）以及警报管理操作（例如静音、学习和确认）。 在传感器控制台中可用。 | - |
| **排除规则** | 根据时间段、设备地址、警报名称或特定传感器指示 IoT，以忽略警报触发器。<br /><br />例如，如果您知道某个特定传感器监视的所有设备设备将经历上午6:30 到10:15 的维护过程，则可以设置一个排除规则，该规则指出此传感器在预定义的时间段内不应发送任何警报。 | **警报 <br /> <br /> 静音警报事件** |

## <a name="h"></a>H

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **地平线开放式开发环境** | 保护 IoT 和 ICS 设备运行与任何标准不相同的专有和自定义协议或协议。 使用范围开放式开发环境 (O) ) SDK 来开发基于定义的协议对网络流量进行解码的 dissector 插件。 用于 IoT 服务的 Defender 分析流量，以提供完整的监视、警报和报告功能。<br /><br />使用 Horizon 可以：<br /><br />- **扩展** 可见性和控制，无需为 IoT 平台版本升级 Defender。<br /><br />- 通过在现场上开发作为外部插件来 **保护** 专有信息。<br /><br />- **本地化** 警报、事件和协议参数的文本。<br /><br />有关详细信息，请联系你的客户成功代表。 | **协议支持 <br /> <br /> 本地化** |
| **范围自定义警报** | 通过为基于范围框架流量 dissectors) 的任何协议 (触发自定义警报，提高企业中的警报管理。<br /><br />这些警报可用于传达信息：<br /><br />-关于基于专用范围插件中的协议和基础协议的通信检测。<br /><br />-关于来自所有协议层的协议字段的组合。 | **协议支持** |

## <a name="i"></a>I

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **集成** | 通过共享设备信息与合作伙伴系统，展开 Defender for IoT 功能。 组织可将以前孤立的安全性、NAC、事件管理和设备管理解决方案桥接起来，以加快系统范围的响应，并更快地降低风险。 | **转发规则** |
| **内部子网** | Defender 为 IoT 定义的子网配置。 在某些情况下（例如使用公共范围作为内部范围的环境），可以指示 IoT 将所有子网解析为内部子网。 子网显示在地图和各种 Defender for IoT 报表中。 | **网上** |

## <a name="s"></a>S

| 术语 | 说明 | 了解详细信息 |
|--|--|--|
| **安全警报** | 处理安全问题的警报，例如过多 SMB 登录尝试或恶意软件检测。 | **警报 <br /> <br /> 操作警报** |
| **选择性探测** | 用于 IoT 的 Defender 被动检查它和流量，并检测有关设备、其属性、行为等的相关信息。 在某些情况下，某些信息可能在被动网络分析中不可见。<br /><br />发生这种情况时，你可以使用 Defender 中的安全、精细的探测工具来发现以前无法访问的设备上的重要信息。 | - |
| **器** | 安装了用于 IoT 平台的 Defender 的物理计算机或虚拟机。 | **本地管理控制台** |
| **网站** | 工厂或其他实体的位置。 站点应包含安装传感器的区域或多个区域。 | **区域** |
| **站点管理** | 用于管理企业传感器的本地管理控制台选项。 | - |
| **智能 IT 学习** | 在学习期间完成并且学习模式处于禁用状态时，使用 IoT 时，IoT 可能会检测到正常 IT 活动（如 DNS 和 HTTP 请求）的结果，这是一个异常的高级基线更改。 此流量可能触发不必要的策略冲突警报和系统通知。 若要减少这些警报和通知，可以启用智能 IT 学习。 | **学习模式 <br /> <br /> 基线** |
| **网上** | 若要在 OT 设备上启用焦点，IT 设备会按子网在设备映射中自动聚合。 每个子网都作为地图上的单个实体提供，其中包括一个交互式折叠或扩展功能，可将精力集中到 IT 子网并返回。 | **设备映射** |
| **系统通知** | 来自本地管理控制台的通知 regrading：<br /><br />-传感器连接状态。<br /><br />-远程备份失败。 | **通知 <br /> <br /> 警报** |
