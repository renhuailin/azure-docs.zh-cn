# <a name="defender-for-iot-glossary-for-organizations"></a>适用于组织的 Defender for IoT 术语表

本术语表简要描述了 Azure Defender for IoT 平台的重要术语。 选择“了解详细信息”链接，转到术语表中的相关术语。 这将有助于你快速了解和使用产品工具。

<a name="glossary-a"></a>

## <a name="a"></a>A

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **访问组** | 通过创建访问组规则来支持大型组织的用户访问要求。<br /><br />规则让你能够控制对 Defender for IoT 本地管理控制台的视图和配置访问，以了解相关业务部门、地区、站点和区域的特定用户角色。<br /><br />例如，允许 Active Directory 组的安全分析师访问西欧汽车数据，但阻止访问非洲的数据。 | **[本地管理控制台](#o)** <br /><br />**[业务部门](#b)** |
| **访问令牌** | 生成访问令牌，以访问 Defender for IoT REST API。 | **[API](#glossary-a)** |
| **确认警报事件** | 指示 Defender for IoT 为检测到的事件隐藏警报一次。 如果再次检测到该事件，将再次触发此警报。 | **[警报](#glossary-a)<br /><br />[了解警报事件](#l)<br /><br />[静音警报事件](#m)** |
| **Alert** | Defender for IoT 引擎触发有关与授权网络行为、网络异常或可疑网络活动和流量的偏差的消息。 | **[转发规则](#f)<br /><br />[排除规则](#e)<br /><br />[系统通知](#s)** |
| **警报注释** | 安全分析人员和管理员在警报消息中的注释。 例如，警报注释可能会提供有关要采取的缓解行动的说明，或有关事件联系的个人姓名。<br /><br />正在查看警报的用户可以选择最能反映事件状态的评论，或调查警报所采取的步骤。 | **[警报](#glossary-a)** |
| **异常引擎** | 用于检测异常计算机到计算机 (M2M) 通信和行为的 Defender for IoT 引擎。 例如，引擎可能检测到过多的 SMB 登录尝试。 检测到这些事件时，将触发异常警报。 | **[Defender for IoT 引擎](#d)** |
| **API** | 允许外部系统访问 Defender for IoT 发现的数据，并通过 SSL 连接使用外部 REST API 执行操作。 | **[访问令牌](#glossary-a)** |
| **攻击途径报表** | 可利用终结点漏洞链的实时图形表示。<br /><br />报表让你能够评估攻击序列中缓解活动的影响以进行确定。 例如，可以评估系统升级是否通过打破攻击链中断攻击者路径，或者是否仍保留备用攻击路径。 这优先处理修正和缓解活动。 | **[风险评估报表](#r)** |

## <a name="b"></a>B

| **业务部门** | 根据特定行业的业务的逻辑组织。<br /><br />例如，包含玻璃厂和塑料厂的全球性公司可以作为两个不同的业务部门进行管理。 可以控制 Defender for IoT 用户对特定业务部门的访问。 | **[本地管理控制台](#o)<br /><br />[访问组](#glossary-a)<br /><br />[站点](#s)<br /><br />[区域](#z)** |
| **Baseline** | 已批准的网络流量、协议、命令和设备。 Defender for IoT 标识与网络基线之间的偏差。 通过生成数据挖掘报表查看已批准的基线流量。 | **[数据挖掘](#d)<br /><br />[学习模式](#l)** |

## <a name="d"></a>D

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **数据挖掘** | 生成有关网络设备的全面和精细报表：<br /><br />- **SOC 事件响应**：以帮助处理即时事件响应的实时报表。 例如，报表可以列出可能需要修补的设备。<br /><br />- **取证**：基于调查报表的历史数据的报表。<br /><br />- **IT 网络完整性**：有助于提高总体网络安全的报表。 例如，报表可以列出身份验证凭据较弱的设备。<br /><br />- **可见性**：涵盖所有查询项目以查看网络所有基线参数的报表。<br /><br />保存数据挖掘报表，供只读用户查看。 | **[基线](#b)<br /><br />[报表](#r)** |
| **Defender for IoT 平台** | 安装在 Defender for IoT 传感器和本地管理控制台上的 Defender for IoT 方案。 | **[传感器](#s)<br /><br />[本地管理控制台](#o)** |
| **设备映射** | Defender for IoT 检测的网络设备的图形表示。 它显示了设备之间的连接和有关每个设备的信息。 使用地图：<br /><br />- 检索和控制关键设备信息。<br /><br />- 分析网络切片。<br /><br />- 导出设备详细信息和摘要。 | **[Purdue 层组](#p)** |
| **设备清单 - 传感器** | 设备清单显示 Defender for IoT 检测到的广泛设备属性。 可用选项如下：<br /><br />- 筛选显示的信息。<br /><br />- 将此信息导出到 CSV 文件。<br /><br />- 导入 Windows 注册表详细信息。 | [组](#g) <br /><br />**[设备清单 - 本地管理控制台](#d)** |
| **设备清单 - 本地管理控制台** | 可从设备清单中的本地管理控制台查看来自连接传感器的设备信息。 这样，本地管理控制台的用户可以全面查看所有网络信息。 | **[设备清单 - 传感器](#d)<br /><br />[设备清单 - 数据集成器](#d)** |
| **设备清单 - 数据集成器** | 本地管理控制台的数据集成功能是你能够利用来自其他企业资源的信息增强设备清单中的数据。 示例资源包括 CMDB、DNS、防火墙和 Web API。 | **[设备清单 - 本地管理控制台](#d)** |

## <a name="e"></a>E

| 术语 | 说明 | 了解更多 |
|--|--|--|
| 引擎 | Defender for IoT 中的自学分析引擎无需更新签名或定义规则。 这些引擎使用特定于 ICS 的行为分析和数据科学来持续分析 OT 网络流量的异常、恶意软件、操作问题、协议违规和基线网络活动的偏差。<br /><br />当引擎检测到偏差时，会触发警报。 可以从“警报”屏幕或从 SIEM 查看和管理警报。 | **[警报](#glossary-a)** |
| **企业视图** | 显示安装 Defenders for IoT 传感器的业务部门、站点和区域的全球地图。 查看恶意警报、操作警报等的地理位置。 | **[业务部门](#b)<br /><br />[站点](#s)<br /><br />[区域](#z)** |
| **事件时间线** | 在网络上检测到的活动的时间线，其中包括：<br /><br />- 触发警报。<br /><br />- 网络事件（信息性）。<br /><br />- 用户操作，如登录、用户删除和用户创建，以及警报管理操作，例如静音、学习和确认。 在传感器控制台中可用。 | - |
| **排除规则** | 指示 Defender for IoT 忽略基于时间段、设备地址和警报名称或特定传感器的警报触发器。<br /><br />例如，如果你知道由特定传感器监视的所有 OT 设备将在早上 6:30 和 10:15 之间完成维护程序，则可以设置排除规则，规定此传感器在预定期间不应发送任何警报。 | **[警报](#glossary-a)<br /><br />[静音警报事件](#m)** |

## <a name="f"></a>F

| 术语 | 说明 | 了解更多 |

## <a name="g"></a>G

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **组** | 包含特定属性的预定义或自定义组设备，例如执行编程活动的设备或位于特定子网中的设备。 使用组来帮助查看设备并分析 Defender for IoT 中的设备。<br /><br />可以在设备映射和设备清单中查看和创建组。 | **[设备映射](#d)<br /><br />[设备清单](#d)** |

## <a name="h"></a>H

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **Horizon 开放式开发环境** | 保护运行专有和自定义协议或非标准协议的 IoT 和 ICS 设备。 使用 Horizon 开放式开发环境 (ODE) SDK，开发析像器插件，用于基于定义的协议解码网络流量。 Defenders for IoT 服务分析流量，以提供完整的监视、警报和报告。<br /><br />使用 Horizon 可以：<br /><br />- 无需升级到 Defender for IoT 平台版本即可扩展可见性和控件。<br /><br />- 通过现场开发以外部插件的形式来保护专有信息。<br /><br />- 将警报、事件和协议参数的文本本地化。<br /><br />有关详细信息，请联系客户成功代表。 | **[协议支持](#p)<br /><br />[本地化](#l)** |
| **Horizon 自定义警报** | 通过触发任何协议（基于 Horizon 框架流量析像器）的自定义警报，加强企业中的警报管理。<br /><br />这些警报可用于传达信息：<br /><br />- 关于基于专有 Horizon 插件中的协议和基础协议的流量检测。<br /><br />- 关于所有协议层的协议字段的组合。 | **[协议支持](#p)** |

## <a name="i"></a>I

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **集成** | 通过与合作伙伴系统共享设备信息来扩展 Defender for IoT 功能。 组织可以连接以前孤立的安全、NAC、事件管理和设备管理解决方案，以加快全系统响应并更快地降低风险。 | **[转发规则](#f)** |
| **内部子网** | Defender for IoT 定义的子网配置。 在某些情况下（例如将公共范围用作内部范围的环境），你可以指示 Defender for IoT 将所有子网作为内部子网进行解析。 子网显示在地图和各种 Defender for IoT 报表中。 | **[子网](#s)** |

| **学习模式** | Defender for IoT 了解你的网络活动时使用的模式。 该活动成为网络的基线。 Defender for IoT 在安装后在预定义的期间内保持在模式中。 在此期间后偏离已学习活动的活动将触发 Defender for IoT 警报。 | **[智能 IT 学习](#s)<br /><br />[基线](#b)** |

| 术语 | 说明 | 了解更多 |

| **区域** | 将全球组织分为地理区域的逻辑区划。 例如北美、西欧和东欧。<br /><br />北美可能具有来自不同业务部门的工厂。 | **[访问组](#glossary-a)<br /><br />[业务部门](#b)<br /><br />[本地管理控制台](#o)<br /><br />[站点](#s)<br /><br />[区域](#z)** |

## <a name="s"></a>S

| 术语 | 说明 | 了解更多 |
|--|--|--|
| **安全警报** | 处理安全问题的警报，例如在尝试或恶意软件检测中过度登录 SMB。 | **[警报操作](#glossary-a)<br /><br />[警报](#o)** |
| **选择性探测** | Defender for IoT 被动地检查 IT 和 OT 流量，并检测设备、其属性、行为等相关信息。 在某些情况下，某些信息在被动网络分析中可能不可见。<br /><br />这种情况发生时，你可以使用 Defender for IoT 中的安全、精细的探测工具来发现以前无法访问设备上的重要信息。 | - |
| **传感器** | 安装了 Defender for IoT 平台的物理或虚拟机。 | **[本地管理控制台](#o)** |
| **站点** | 工厂或其他实体的位置。 站点应包含安装传感器的区域或多个区域。 | **[区域](#z)** |
| **站点管理** | 让你能够管理企业传感器的本地管理控制台选项。 | - |
| **智能 IT 学习** | 在学习期满和学习模式被禁用后，Defender for IoT 可能会检测到由于正常 IT 活动（如 DNS 和 HTTP 请求）而产生的异常高级基线变化。 此流量可能会触发不必要的策略违规警报和系统通知。 若要减少这些警报和通知，可以启用智能 IT 学习。 | **[学习模式](#l)<br /><br />[基线](#b)** |
| **子网** | 若要实现对 OT 设备的聚焦，IT 设备通过设备映射中的子网自动聚合。 每个子网在地图上显示为单个实体，包括交互式折叠或扩展功能，以聚焦到 IT 子网并返回。 | **[设备映射](#d)** |
| **系统通知** | 来自本地管理控制台的通知与以下内容有关：<br /><br />- 传感器连接状态。<br /><br />- 远程备份失败。 | **[通知](#n)<br /><br />[警报](#glossary-a)** |
